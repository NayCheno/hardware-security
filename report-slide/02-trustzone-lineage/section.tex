\begin{frame}[t,shrink=6]{Arm TrustZone 白皮书：研究背景}
\DeckKicker{研究背景}
\SlideMeta{arm\_trustzone\_whitepaper}{E4 / Foundational industry evidence}{local\_pdf\_verified}
{\large\bfseries\color{Ink} TrustZone 的背景问题是移动和嵌入式设备需要把小型可信软件栈从通用 OS 中隔离出来。\par}\vspace{1.8mm}
\begin{columns}[T,totalwidth=\textwidth]
  \begin{column}{0.45\textwidth}
    \fcolorbox{Rule}{Paper}{%
  \begin{minipage}[t]{0.97\linewidth}
    \vspace{1.1mm}
    \hspace{1.4mm}\begin{minipage}{0.92\linewidth}
      \PanelTitle{讲解逻辑}
      \scriptsize \begin{itemize}
  \item 早期设备需要保护密钥、DRM、支付、secure boot 和设备管理功能。
  \item 通用 OS 代码量大、可扩展性强，也更容易被攻击，因此需要一个硬件支持的隔离世界。
  \item TrustZone 的设计目标是让 secure services 与 rich OS 并存，而不是替代整个 OS。
  \item 这解释了为什么 TrustZone TCB 往往包含 secure OS、trusted app 和 vendor services。
\end{itemize}

\begin{frame}[t,shrink=6]{Arm TrustZone 白皮书：解决方案}
\DeckKicker{解决方案}
\SlideMeta{arm\_trustzone\_whitepaper}{E4 / Foundational industry evidence}{local\_pdf\_verified}
{\large\bfseries\color{Ink} TrustZone 的解决方案是安全状态传播，而不是为每个应用创建独立硬件 enclave。\par}\vspace{1.8mm}
\begin{columns}[T,totalwidth=\textwidth]
  \begin{column}{0.45\textwidth}
    \fcolorbox{Rule}{Paper}{%
  \begin{minipage}[t]{0.97\linewidth}
    \vspace{1.1mm}
    \hspace{1.4mm}\begin{minipage}{0.92\linewidth}
      \PanelTitle{讲解逻辑}
      \scriptsize \begin{itemize}
  \item 处理器、总线和外设通过 secure/non-secure 属性区分访问。
  \item secure monitor 负责 world switch，Normal World 通过 SMC 等接口请求 secure service。
  \item 内存和外设访问控制通常由 SoC 级组件完成。
  \item 这也是它与后续 Realm/TVM 的关键区别：TrustZone 更像双世界模型，不是多租户 confidential VM 模型。
\end{itemize}
    \end{minipage}
    \vspace{1mm}
  \end{minipage}}
  \end{column}
  \begin{column}{0.49\textwidth}
    \fcolorbox{Rule}{Panel}{%
  \begin{minipage}[t]{0.97\linewidth}
    \vspace{1.1mm}
    \hspace{1.4mm}\begin{minipage}{0.92\linewidth}
      \PanelTitle{两种隔离抽象}
      \scriptsize \textbf{TrustZone}\par
Secure World → Normal World → secure monitor → secure services\par
{\tiny\textcolor{Muted}{双世界隔离，常用于平台服务}}\par\vspace{1.8mm}
\textbf{CCA/CoVE}\par
Realm/TVM → untrusted host → RMM/TSM → per-VM lifecycle\par
{\tiny\textcolor{Muted}{面向多租户 confidential workload}}\par\vspace{1.8mm}
    \end{minipage}
    \vspace{1mm}
  \end{minipage}}
  \end{column}
\end{columns}
\vspace{1mm}
{\tiny\textcolor{Muted}{Source: 对比基于 TrustZone 白皮书机制和本报告 CCA/CoVE taxonomy。}}
\end{frame}
