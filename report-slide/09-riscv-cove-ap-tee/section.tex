\begin{frame}[t,shrink=6]{RISC-V CoVE reference architecture：研究背景}
\DeckKicker{研究背景}
\SlideMeta{sahita2023cove}{E3 / Foundational public preprint}{local\_pdf\_verified}
{\large\bfseries\color{Ink} CoVE 的背景问题是传统 RISC-V privilege model 默认 machine-mode firmware 可信，但云场景要把 host/VMM 排除出 tenant TCB。\par}\vspace{1.8mm}
\begin{columns}[T,totalwidth=\textwidth]
  \begin{column}{0.45\textwidth}
    \fcolorbox{Rule}{Paper}{%
  \begin{minipage}[t]{0.97\linewidth}
    \vspace{1.1mm}
    \hspace{1.4mm}\begin{minipage}{0.92\linewidth}
      \PanelTitle{讲解逻辑}
      \scriptsize \begin{itemize}
  \item RISC-V machine mode 权限最高，平台 firmware 通常天然可信。
  \item 多租户云平台中，host OS、VMM 和平台管理软件都可能不被 tenant 信任。
  \item enclave framework 保护进程或应用，不能直接表达完整 VM lifecycle。
  \item CoVE 因此把 RISC-V TEE 问题推进到 TVM 抽象。
\end{itemize}
    \end{minipage}
    \vspace{1mm}
  \end{minipage}}
  \end{column}

\textbf{维度} & \textbf{pre-CoVE enclave} & \textbf{CoVE TVM} \\

\begin{frame}[t,shrink=6]{RISC-V CoVE reference architecture：解决方案}
\DeckKicker{解决方案}
\SlideMeta{sahita2023cove}{E3 / Foundational public preprint}{local\_pdf\_verified}
{\large\bfseries\color{Ink} CoVE 的解决方案是让 TSM 成为 TVM lifecycle 的 gatekeeper，并通过 SBI/ABI 暴露受控管理接口。\par}\vspace{1.8mm}
\begin{columns}[T,totalwidth=\textwidth]
  \begin{column}{0.45\textwidth}
    \fcolorbox{Rule}{Paper}{%
  \begin{minipage}[t]{0.97\linewidth}
    \vspace{1.1mm}
    \hspace{1.4mm}\begin{minipage}{0.92\linewidth}
      \PanelTitle{讲解逻辑}
      \scriptsize \begin{itemize}
  \item host 请求创建 TVM、添加 pages 和 vCPUs，但关键状态转换由 TSM 检查。
  \item TVM 内部通过 guest ABI 请求受信服务。
  \item confidential memory 与普通 memory 的转换成为 lifecycle 核心。
  \item 这种结构与 Arm CCA/RMM 在理念上可对照，但具体接口不同。
\end{itemize}
    \end{minipage}
    \vspace{1mm}
  \end{minipage}}
  \end{column}
  \begin{column}{0.49\textwidth}
    \fcolorbox{Rule}{Panel}{%

    \vspace{1.1mm}
    \hspace{1.4mm}\begin{minipage}{0.92\linewidth}
      \PanelTitle{host path vs TVM path}
      \scriptsize \textbf{host management}\par
VMM request → TSM check → TVM resource update → run TVM\par
{\tiny\textcolor{Muted}{host manages but is mediated}}\par\vspace{1.8mm}
\textbf{TVM service}\par
guest request → confidential ABI → TSM service → attestation/memory response\par
{\tiny\textcolor{Muted}{tenant path avoids trusting host}}\par\vspace{1.8mm}
    \end{minipage}
    \vspace{1mm}
