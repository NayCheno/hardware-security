\PaperStory
{RISC-V CoVE reference architecture}
{\Meta{sahita2023cove}{Foundational / E1 arXiv 2023}{source: local PDF verified}{claim: reference architecture}}
{\begin{itemize}
  \item CoVE reference architecture 提出 RISC-V confidential computing 参考架构方向。
  \item 它明确 confidential VM 需要 ISA、non-ISA、SoC 与 platform requirements 的配合。
\end{itemize}}
{\begin{itemize}
  \item 多租户 confidential VM 需要降低 tenant 对 host/hypervisor 的信任。
  \item 该问题不同于单进程 enclave lineage，需要 TVM、platform 和 attestation 共同定义。
\end{itemize}}
{\begin{itemize}
  \item 核心洞察是把 confidential VM 从普通虚拟化和 PMP enclave 中拆出来。
  \item 解决方案用 CoVE reference architecture 描述 TVM 所需的硬件、固件、hypervisor 和 platform support。
\end{itemize}}
{\begin{itemize}
  \item RISC-V confidential VM reference architecture。
  \item ISA/non-ISA/SoC/platform requirement split。
  \item Host/hypervisor trust reduction for TVM。
\end{itemize}}
{\begin{itemize}
  \item 架构论文/position-style，无完整系统实验。
  \item 具体 ABI、状态机和 lifecycle details 需要 AP-TEE draft 支撑。
\end{itemize}}
{\begin{itemize}
  \item \textbf{优点：}CoVE confidential VM 入口材料，把 RISC-V 从 enclave lineage 推向 TVM 模型。
  \item \textbf{不足：}不提供完整 ABI、状态机或 ratified standard 语义。
  \item \textbf{商业落地：}有助开放 confidential VM 生态；风险在标准未定、平台支持和 verifier ecosystem。
\end{itemize}}

\PaperStory
{RISC-V AP-TEE / CoVE TVM 规范草案}
{\Meta{riscv\_ap\_tee\_2024}{Draft/not ratified / E3 AP-TEE v0.7 RC2}{source: local PDF verified}{claim: draft lifecycle and ABI semantics}}
{\begin{itemize}
  \item AP-TEE draft 定义 RISC-V AP-TEE/CoVE TVM lifecycle 和 SBI ABI。
  \item 它覆盖 TVM、TSM、COVH/COVG 与 memory lifecycle 等核心对象。
\end{itemize}}
{\begin{itemize}
  \item CoVE 需要标准化 TVM、TSM、memory lifecycle、attestation 和 host/guest 调用界面。
  \item 这些语义让 RISC-V AP-TEE 能与 Arm CCA/RMM 在同层比较。
\end{itemize}}
{\begin{itemize}
  \item 核心洞察是把 TVM lifecycle 和 host/guest ABI 变成可实现、可验证的规范对象。
  \item 解决方案给出 TSM、TSM-driver、Supervisor Domains、COVH/COVG、memory donation/reclaim/share 等 draft 语义。
\end{itemize}}
{\begin{itemize}
  \item TVM/TSM lifecycle and SBI ABI。
  \item COVH/COVG host/guest interface split。
  \item Memory donation, reclaim and sharing semantics。
\end{itemize}}
{\begin{itemize}
  \item 规范草案，无实验。
  \item Draft/not-ratified 状态下不能写成 ratified standard 或 production ABI。
\end{itemize}}
{\begin{itemize}
  \item \textbf{优点：}当前最关键的 RISC-V CCA 对照材料，直接定义 TVM/TSM lifecycle。
  \item \textbf{不足：}v0.7 RC2 仍是 draft，状态机和 ABI 可能变化。
  \item \textbf{商业落地：}潜力在开放 confidential VM stack；风险在标准收敛、firmware/OS support 和兼容迁移。
\end{itemize}}
