\PaperStory
{Shelter: CCA 用户态隔离}
{\Meta{zhang2023shelter}{Foundational / E1 USENIX Security 2023}{source: local PDF verified}{claim: prototype and overhead claims from primary paper}}
{\begin{itemize}
  \item Shelter 用 Arm CCA 支持用户态隔离，把 Realm 能力从 VM 级机密计算推进到 user-space compartment。
  \item 本 deck 用它作为 CCA 细粒度隔离方向入口。
\end{itemize}}
{\begin{itemize}
  \item Realm 机制不必只服务 VM；进程内敏感代码同样需要小 TCB、host-untrusted 的隔离边界。
  \item 传统进程或库级隔离仍可能依赖过大的 OS/runtime 信任基础。
\end{itemize}}
{\begin{itemize}
  \item 核心洞察是把 CCA primitives 封装成可被应用使用的隔离抽象。
  \item 解决方案让应用 compartment 在不完全信任 OS/host 的前提下运行敏感逻辑。
\end{itemize}}
{\begin{itemize}
  \item User-space compartment over Arm CCA。
  \item Realm-backed isolation boundary。
  \item Runtime and OS integration path for fine-grained protection。
\end{itemize}}
{\begin{itemize}
  \item USENIX Security 论文提供原型与开销评估。
  \item 具体数值需回引本地 PDF；这里不从二手摘要复制性能数字。
\end{itemize}}
{\begin{itemize}
  \item \textbf{优点：}开辟 CCA 细粒度使用方式，是从 VM Realm 走向应用隔离的方向入口。
  \item \textbf{不足：}系统原型不等同于通用 ABI 或量产部署，生态依赖 runtime、OS 与 CCA 硬件支持。
  \item \textbf{商业落地：}适合云端敏感函数、插件隔离和移动端安全服务；风险在开发者可用性和平台普及速度。
\end{itemize}}

\PaperStory
{RContainer: CCA 安全容器}
{\Meta{zhou2025rcontainer}{Peer-reviewed SOTA / E1 NDSS 2025}{source: local PDF verified}{claim: mechanism and evaluation claims from primary paper}}
{\begin{itemize}
  \item RContainer 把 Arm CCA 扩展到安全容器架构，面向云原生场景重构容器隔离边界。
  \item 本 deck 用它代表 CCA 从 confidential VM 走向容器 workload 的落地点。
\end{itemize}}
{\begin{itemize}
  \item 容器共享内核带来部署效率，但传统容器边界难以抵御高权限 host/内核路径的泄露风险。
  \item 云原生用户希望保留容器工作流，同时获得更接近 confidential VM 的机密性边界。
\end{itemize}}
{\begin{itemize}
  \item 核心洞察是把容器 runtime/lifecycle 与 CCA 保护域 lifecycle 对齐。
  \item 解决方案利用 CCA 硬件 primitives 为容器 workload 建立更强机密性边界。
\end{itemize}}
{\begin{itemize}
  \item CCA-backed secure container boundary。
  \item Container lifecycle and Realm lifecycle mapping。
  \item Runtime/orchestration integration considerations。
\end{itemize}}
{\begin{itemize}
  \item NDSS 系统论文提供原型评估。
  \item 本 deck 不复制未核对的具体性能数字；production readiness 仍需云平台和编排系统证据。
\end{itemize}}
{\begin{itemize}
  \item \textbf{优点：}贴近云原生落地，把 CCA 从 confidential VM 推向容器 workload。
  \item \textbf{不足：}容器生态、编排系统和 CCA lifecycle 的耦合复杂。
  \item \textbf{商业落地：}适合云平台安全容器服务；风险在 Kubernetes/runtime 改造、调试可观测性和租户迁移成本。
\end{itemize}}

\PaperStory
{NanoZone: CCA 细粒度内存保护}
{\Meta{liu2025nanozone}{Draft/not ratified / E3 arXiv 2025}{source: local PDF verified}{claim: draft evidence only}}
{\begin{itemize}
  \item NanoZone 探索 Arm CCA 上更可扩展、更细粒度的内存保护域。
  \item 本 deck 用它代表 CCA 粒度从 VM/container 继续下探的前沿方向。
\end{itemize}}
{\begin{itemize}
  \item 只用 Realm/VM 粒度会在管理成本、保护域数量和应用内敏感数据隔离之间形成矛盾。
  \item 更小保护域需要重新审视 CCA granule、lifecycle 和管理者信任边界。
\end{itemize}}
{\begin{itemize}
  \item 核心洞察是利用 CCA granule/lifecycle 支撑更小保护域。
  \item \Key{liu2025lesstrust} 作为辅助材料，对照 intra-process isolation 和 less-trust management threat model。
\end{itemize}}
{\begin{itemize}
  \item Fine-grained memory protection over CCA。
  \item Granule/lifecycle scaling pressure。
  \item Intra-process isolation comparison with less-trust management。
\end{itemize}}
{\begin{itemize}
  \item arXiv 预印本提供设计与评估线索。
  \item Draft/not ratified 状态下只作为前沿方向证据，不能写成同行评议共识或成熟生产证据。
\end{itemize}}
{\begin{itemize}
  \item \textbf{优点：}直接回应 CCA 保护粒度和可扩展性问题。
  \item \textbf{不足：}预印本尚不能代表同行评议共识或硬件/OS 生态成熟度。
  \item \textbf{商业落地：}可服务插件沙箱、密钥处理和多租户组件隔离；风险在 ABI 稳定性和开发复杂度。
\end{itemize}}
