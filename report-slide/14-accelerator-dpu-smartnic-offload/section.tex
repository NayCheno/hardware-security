\begin{frame}[t,shrink=6]{HETEE: rack-scale heterogeneous TEE：内容摘要}

\begin{columns}[T,totalwidth=\textwidth]

      \scriptsize \begin{itemize}
  \item 论文指出大规模 confidential computing 受限于 GPU/FPGA/TPU 等高吞吐 accelerator 缺乏可扩展保护。
  \item HETEE 不要求修改 CPU/GPU chips，而是用 security controller 和资源池化组织可信执行。
  \item 它支持 elastic resource allocation，使多个 accelerator 可服务一个 enclave-like task。
  \item 它是 accelerator confidential computing 的 foundational systems paper。
\end{itemize}

      \scriptsize \begin{itemize}
  \item 01. tenant submits CDI task
  \item 02. Security Controller manages keys/policy
  \item 03. elastic accelerator resources allocated
  \item 04. data encrypted across untrusted fabric
  \item 05. accelerators execute protected workload
  \item 06. results returned to tenant

\SlideMeta{zhu2020hetee}{E1 / Foundational peer-reviewed}{local\_pdf\_verified}
{\large\bfseries\color{Ink} HETEE 的背景问题是 CPU TEE 无法承担计算密集型 workloads，而 GPU/FPGA 又通常没有同等可信边界。\par}\vspace{1.8mm}

      \scriptsize \begin{itemize}
  \item AI training/inference 和 data-intensive computing 需要高吞吐 accelerators。
  \item 把敏感数据留在 CPU enclave 会失去 accelerator 性能价值。
  \item 直接把数据发给 accelerator 又会暴露给 host、fabric、device runtime。
  \item HETEE 选择在 rack 级引入 security controller，而不是等待每个 accelerator 原生 TEE。
\end{itemize}

\begin{frame}[t,shrink=6]{HETEE: rack-scale heterogeneous TEE：解决方案}
\DeckKicker{解决方案}
\SlideMeta{zhu2020hetee}{E1 / Foundational peer-reviewed}{local\_pdf\_verified}
{\large\bfseries\color{Ink} HETEE 的设计把 trusted control plane 收缩到 Security Controller，把重计算留给现有 accelerators。\par}\vspace{1.8mm}
\begin{columns}[T,totalwidth=\textwidth]
  \begin{column}{0.45\textwidth}
    \fcolorbox{Rule}{Paper}{%
  \begin{minipage}[t]{0.97\linewidth}
    \vspace{1.1mm}
    \hspace{1.4mm}\begin{minipage}{0.92\linewidth}
      \PanelTitle{讲解逻辑}
      \scriptsize \begin{itemize}
  \item SC 包含 RNG、KDF、MAC 等安全功能模块。
  \item 资源隔离和密钥管理由 SC 统一协调。
  \item 计算单元可弹性分配给 protected task。
  \item 这种架构降低 chip-level 修改要求，但把 SC 变成关键 TCB。
\end{itemize}
    \end{minipage}

  \item 01. authenticate task
  \item 02. derive keys
  \item 03. assign accelerator resources
  \item 04. isolate data path
  \item 05. verify completion
