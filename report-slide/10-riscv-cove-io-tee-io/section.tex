\section{RISC-V CoVE-IO / TEE-I/O}

\DirectionIntro
{RISC-V CoVE-IO / TEE-I/O}
{本方向把设备身份、DMA/MMIO、interrupt、link security 与 I/O translation 放到同一边界，说明 confidential VM 要安全使用真实设备。}
{三篇主讲材料是 \Key{feng2024siopmp}、\Key{riscv_cove_io_2026}、\Key{riscv_iommu_2023}；\Key{riscv_aia_2023} 只作为 trusted interrupt auxiliary。}
{CoVE-IO 必须保留 draft/not-ratified 标签；IOMMU/AIA 是 substrate，不是完整 confidential I/O 方案。}

\PaperStory
{sIOPMP: scalable I/O protection for TEE}
{\Meta{feng2024siopmp}{Peer-reviewed SOTA / E1 ASPLOS 2024}{source: local PDF verified}{claim: scalable I/O access-control mechanism}}
{\begin{itemize}
  \item sIOPMP 提出面向 TEE 的 scalable I/O protection。
  \item 它聚焦设备 access control 和 metadata/checking 可扩展性。
\end{itemize}}
{\begin{itemize}
  \item DMA 设备可绕过 CPU memory isolation。
  \item 只保护 CPU 页表或 PMP 不足以覆盖设备侧访问，I/O 保护成为 TEE 的关键缺口。
\end{itemize}}
{\begin{itemize}
  \item 核心洞察是把设备访问控制做成可扩展 metadata/checking 路径。
  \item 解决方案提高设备访问控制效率，并支撑更大规模 I/O protection。
\end{itemize}}
{\begin{itemize}
  \item Scalable I/O protection metadata。
  \item Device access-control checking path。
  \item TEE-oriented DMA isolation substrate。
\end{itemize}}
{\begin{itemize}
  \item ASPLOS 论文给出吞吐和开销评估。
  \item 具体数值以本地 PDF 为准；本页不复制未复核的 benchmark。
\end{itemize}}
{\begin{itemize}
  \item \textbf{优点：}RISC-V I/O protection 的 peer-reviewed SOTA，可作为 CoVE-IO access-control 前置材料。
  \item \textbf{不足：}不覆盖完整 device identity、TDISP、link security 或 trusted interrupt path。
  \item \textbf{商业落地：}对高性能设备直通有价值；风险在硬件改造、IOMMU/IOPMP 组合和平台可观测性。
\end{itemize}}

\PaperStory
{RISC-V CoVE-IO 规范草案}
{\Meta{riscv\_cove\_io\_2026}{Draft/not ratified / E3 CoVE-IO v0.3.0}{source: local PDF verified}{claim: draft confidential I/O lifecycle}}
{\begin{itemize}
  \item CoVE-IO draft 定义 RISC-V confidential I/O 架构。
  \item 它把 TVM 设备使用扩展到 device identity、DMA/MMIO、interrupt 和 link security。
\end{itemize}}
{\begin{itemize}
  \item TVM 使用真实设备时，安全边界必须覆盖设备身份、DMA/MMIO、interrupt delivery、link protection 和管理 control plane。
  \item 这些对象跨 CPU、IOMMU、device、PCIe/CXL link 和 verifier。
\end{itemize}}
{\begin{itemize}
  \item 核心洞察是把 trusted device lifecycle 放入 TVM lifecycle。
  \item 解决方案用 TDI/TDM/DSM、SPDM、TDISP、PCIe IDE、trusted MSI 等机制组合描述 draft confidential I/O。
\end{itemize}}
{\begin{itemize}
  \item TDI/TDM/DSM device-domain vocabulary。
  \item SPDM/TDISP and PCIe IDE linkage。
  \item Trusted MSI and interrupt-path concepts。
\end{itemize}}
{\begin{itemize}
  \item 规范草案，无实验。
  \item Draft/not-ratified 状态下不能写成 ratified standard 或 production CoVE-IO stack。
\end{itemize}}
{\begin{itemize}
  \item \textbf{优点：}当前最重要的 RISC-V confidential I/O 规范入口。
  \item \textbf{不足：}v0.3.0 draft 仍可能变化，不证明任一硬件或 OS 实现成熟。
  \item \textbf{商业落地：}对 RISC-V confidential accelerator/NIC 生态关键；风险在 SPDM/TDISP/IDE 与平台 ABI 收敛。
\end{itemize}}

\PaperStory
{RISC-V IOMMU 支撑 CoVE-IO}
{\Meta{riscv\_iommu\_2023}{Spec-standard SOTA / E0}{source: local PDF verified}{claim: I/O translation/protection semantics}}
{\begin{itemize}
  \item RISC-V IOMMU 支撑设备 DMA translation/protection。
  \item 它是 CoVE-IO 设备侧地址访问控制的标准底座。
\end{itemize}}
{\begin{itemize}
  \item CoVE-IO 不能只靠 PMP 或 CPU page table。
  \item 设备侧 DMA 需要独立的 translation/protection 语义，才能限制 device 对 TVM/host memory 的访问。
\end{itemize}}
{\begin{itemize}
  \item 核心洞察是由 IOMMU 控制设备可访问地址空间。
  \item \Key{riscv_aia_2023} 只作为 trusted MSI/interrupt delivery 辅助材料。
\end{itemize}}
{\begin{itemize}
  \item Device DMA translation/protection。
  \item I/O page-table permission semantics。
  \item Integration point with CoVE-IO trusted device lifecycle。
\end{itemize}}
{\begin{itemize}
  \item 规范，无实验。
  \item 只支撑公开 I/O translation/protection 语义，不证明完整 confidential I/O。
\end{itemize}}
{\begin{itemize}
  \item \textbf{优点：}CoVE-IO 中 DMA isolation 和 I/O translation 的标准支撑点。
  \item \textbf{不足：}IOMMU 不是单独的 confidential I/O 方案，也不覆盖 device identity、SPDM/TDISP 或 link security。
  \item \textbf{商业落地：}适合服务器设备直通与多租户 DMA 管控；风险在配置完整性、固件测量和 verifier 可见性。
\end{itemize}}

\DirectionSummary
{RISC-V CoVE-IO / TEE-I/O}
{sIOPMP 解决 scalable I/O access control，但范围仍小于完整 confidential I/O。}
{CoVE-IO draft 把 TDI/TDM/DSM、SPDM/TDISP、PCIe IDE 和 trusted MSI 纳入 TVM device lifecycle。}
{IOMMU 提供 DMA translation/protection 标准 substrate；AIA 作为 interrupt auxiliary 补充 trusted MSI 语义。}
{下一步需要在 13-15 中把这些规范对象映射到 SPDM/TDISP、vendor NIC/DPU attestation 和 secure vNIC/vSwitch/offload production evidence。}
