{\large\bfseries\color{Ink} sIOPMP 的核心贡献是把 RISC-V I/O protection 做成可扩展、可热插拔、适合 TEE 的 DMA 访问控制。\par}\vspace{1.8mm}

      \scriptsize \begin{itemize}
  \item 论文指出现有 TEE 常把 I/O isolation 当成二等问题，恶意 DMA 可绕过 CPU-side checks。
  \item 传统 IOMMU 在 TEE 场景中可能引入大 TCB、flush 成本和 page-granularity 不匹配。
  \item sIOPMP 提出 multi-stage-tree checker、mountable entries、ID remapping 和 ownership interface。

\begin{frame}[t,shrink=6]{sIOPMP: scalable I/O protection for TEE：研究背景}
\DeckKicker{研究背景}
\SlideMeta{feng2024siopmp}{E1 / Peer-reviewed SOTA}{local\_pdf\_verified}
{\large\bfseries\color{Ink} sIOPMP 的背景问题是 TEE 需要 small TCB 和 region-based I/O isolation，而通用 IOMMU 不总是合适。\par}\vspace{1.8mm}
\begin{columns}[T,totalwidth=\textwidth]
  \begin{column}{0.45\textwidth}
    \fcolorbox{Rule}{Paper}{%
  \begin{minipage}[t]{0.97\linewidth}
    \vspace{1.1mm}
    \hspace{1.4mm}\begin{minipage}{0.92\linewidth}
      \PanelTitle{讲解逻辑}
      \scriptsize \begin{itemize}
  \item CPU-side PMP 不能阻止 device 通过 DMA 访问 secure memory。
  \item IOMMU 可以做 DMA translation，但把复杂 IOMMU 放入 TCB 会增加信任面。
  \item IOMMU page-level isolation 与 TEE region-based policy 可能失配。
  \item TEE 还要支持热插拔、虚拟设备和大量 memory regions。
\end{itemize}

\textbf{问题} & \textbf{传统 IOMMU 风险} & \textbf{sIOPMP 方向} \\

\begin{frame}[t,shrink=6]{sIOPMP: scalable I/O protection for TEE：解决方案}
\DeckKicker{解决方案}
\SlideMeta{feng2024siopmp}{E1 / Peer-reviewed SOTA}{local\_pdf\_verified}
{\large\bfseries\color{Ink} sIOPMP 用 mountable IOPMP、ID remapping 和 ownership interface，把 I/O isolation 变成 TEE 可管理资源。\par}\vspace{1.8mm}
\begin{columns}[T,totalwidth=\textwidth]
  \begin{column}{0.45\textwidth}
    \fcolorbox{Rule}{Paper}{%
  \begin{minipage}[t]{0.97\linewidth}
    \vspace{1.1mm}
    \hspace{1.4mm}\begin{minipage}{0.92\linewidth}
      \PanelTitle{讲解逻辑}
      \scriptsize \begin{itemize}
  \item mountable IOPMP 支持 unlimited number of devices。
  \item ID remapping 通过 CAM table 记录 device ID 到 SID 的映射。
  \item microarchitecture 使用 multi-stage-tree checker 和 packet masking。
  \item ownership-based interface 让 TEE runtime 以资源所有权方式更新 I/O policy。
\end{itemize}
    \end{minipage}
    \vspace{1mm}
  \end{minipage}}

      \PanelTitle{sIOPMP policy update}
      \scriptsize \begin{itemize}
  \item 01. TEE assigns device ownership
  \item 02. update ID-to-SID mapping
  \item 03. mount region entries
  \item 04. checker evaluates DMA
  \item 05. violation masked or blocked
