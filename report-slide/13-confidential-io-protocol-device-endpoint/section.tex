\PaperFrame
{SPDM device identity and measurement}
{dmtf\_spdm\_2025 + dmtf\_spdm\_arch\_2024 | Foundational / DMTF spec + white paper}
{定义 SPDM 的 device identity、measurement、capability negotiation、secured session 和 architecture role。}
{confidential workload 使用真实设备时，host 对设备的普通枚举结果不足以作为可信证据。}
{通过 SPDM 让 requester 获取 responder 身份、测量和安全会话能力；secured messages 与 SPDM over TCP 补充会话保护和传输绑定。}
{规范/白皮书，无新实验；证据类型是标准化接口和协议语义。}
{适合作为机密计算 device identity/evidence 层基础；不能单独证明 DMA、interrupt 或 link path 已安全。}

\PaperFrame
{CoVE-IO 与 TDISP/TDISP XT}
{riscv\_cove\_io\_2026 + pcisig\_tdisp\_2022 + pcisig\_tdisp\_xt\_2025 | Draft + gated metadata}
{CoVE-IO 把 TVM trusted I/O 拆成 TDI/TDM/DSM、SPDM、TDISP、PCIe IDE、IOMMU、trusted MSI 等组合。}
{TVM/Realm 使用设备时，需要证明设备接口身份、绑定 interface state、保护 DMA/MMIO，并把 interrupt delivery 纳入同一边界。}
{用 device-interface lifecycle、设备安全管理、link protection 和 I/O translation 协同构造 confidential I/O boundary。}
{规范草案/ECN metadata，无实验；TDISP/TDISP XT public PDFs 不可直接下载。}
{是本 survey 的 RISC-V confidential I/O 主锚点；必须明确标准状态，避免写成已完全落地规范。}

\PaperFrame
{trusted network/offload tradeoff}
{li2024folio + amd\_sev\_tio\_2023 + weinhold2025tlsra | Peer-reviewed SOTA + industry + protocol evidence}
{FOLIO 研究不信任 I/O device 时 confidential VM 的高性能网络路径；SEV-TIO 展示 VM trusted I/O 产业设计；TLS+RA 绑定 attested TEE 与 TLS channel。}
{网络和 accelerator offload 会把 plaintext、queue state、keys、model state 或 TLS private key 带出 CPU confidential boundary。}
{分别代表三种路线：避免信任设备、把设备纳入 trusted I/O boundary、以及用 attested TLS 把 application channel 与 TEE evidence 绑定。}
{FOLIO 和 TLS+RA 为系统/协议论文，有原型评估；SEV-TIO 为 white paper，无独立学术实验。}
{适合写 trusted network/offload tradeoff；AMD 和 TLS+RA 材料只作 cross-platform evidence。}
